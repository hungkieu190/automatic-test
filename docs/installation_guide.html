Selenium Testing Framework - Installation and Usage Guide

This guide provides instructions for installing and using the Selenium testing framework for the Washbank Kiosk project on Ubuntu, macOS, and Windows platforms.

Note: This framework uses Python with Selenium WebDriver to automate browser testing for web applications.

Prerequisites

Before installing the testing framework, ensure you have the following prerequisites:

  • Python 3.8 or higher
  • Pip (Python package installer)
  • Google Chrome or Firefox browser installed
  • Internet connection (for downloading WebDriver)

Installation Instructions

Ubuntu Installation

Step 1: Install Python and pip

sudo apt update
sudo apt install python3 python3-pip python3-venv

Step 2: Create a virtual environment

mkdir -p ~/devland/automatic-test/thimpress
cd ~/devland/automatic-test/thimpress
python3 -m venv .

Step 3: Activate the virtual environment

source bin/activate

Step 4: Install required packages

pip install selenium webdriver-manager pytest pytest-html

Step 5: Install Chrome browser (if not already installed)

sudo apt update
sudo apt install google-chrome-stable
Note: The webdriver-manager package will automatically download and manage the appropriate ChromeDriver version for your Chrome browser.

macOS Installation

Step 1: Install Homebrew (if not already installed)

/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Step 2: Install Python

brew install python

Step 3: Create a project directory and virtual environment

mkdir -p ~/devland/automatic-test/thimpress
cd ~/devland/automatic-test/thimpress
python3 -m venv .

Step 4: Activate the virtual environment

source bin/activate

Step 5: Install required packages

pip install selenium webdriver-manager pytest pytest-html

Step 6: Install Chrome browser (if not already installed)

brew install --cask google-chrome
Note: The webdriver-manager package will automatically download and manage the appropriate ChromeDriver version for your Chrome browser.

Windows Installation

Step 1: Install Python

Download and install Python from python.org. Make sure to check "Add Python to PATH" during installation.

Step 2: Create a project directory and virtual environment

mkdir C:\devland\automatic-test\thimpress
cd C:\devland\automatic-test\thimpress
python -m venv .

Step 3: Activate the virtual environment

.\Scripts\activate

Step 4: Install required packages

pip install selenium webdriver-manager pytest pytest-html

Step 5: Install Chrome browser (if not already installed)

Download and install Google Chrome from google.com/chrome

Note: The webdriver-manager package will automatically download and manage the appropriate ChromeDriver version for your Chrome browser.

Usage Guide

Basic Usage

After installation, you can run the existing test scripts or create your own.

Running the Washbank Kiosk Test

cd ~/devland/automatic-test/thimpress  # Adjust path for Windows if needed
source bin/activate  # On Windows: .\Scripts\activate
pytest test_washbank_kiosk.py -v -s

Running the Basic Selenium Test

python test_selenium.py

Advanced Usage

The testing framework supports various options for running tests:

Running Tests with Debug Mode

pytest test_washbank_kiosk.py -v -s --pdb

The --pdb option will stop at the point of failure for debugging.

Generating HTML Reports

pytest test_washbank_kiosk.py -v --html=report.html

This will generate an HTML report of the test results in the current directory.

Running Specific Test Methods

pytest test_washbank_kiosk.py::TestWashbankKiosk::test_check_dashboard -v

Common Commands

Command Description
pytest -v Run tests with verbose output
pytest -s Show print statements during test execution
pytest --pdb Enter debugger on test failures
pytest -k "keyword" Run tests that match the keyword expression
pytest --html=report.html Generate HTML report of test results

Creating Your Own Tests

To create your own test scripts, follow this basic structure:

import pytest
from selenium import webdriver
from selenium.webdriver.chrome.service import Service
from webdriver_manager.chrome import ChromeDriverManager
from selenium.webdriver.common.by import By

class TestYourWebsite:
    @classmethod
    def setup_class(cls):
        cls.service = Service(ChromeDriverManager().install())
        cls.driver = webdriver.Chrome(service=cls.service)
        cls.driver.implicitly_wait(10)
        
    def test_your_feature(self):
        self.driver.get("https://your-website-url.com")
        # Add your test assertions here
        
    @classmethod
    def teardown_class(cls):
        cls.driver.quit()

Troubleshooting

Common Issues

1. WebDriver Manager fails to download driver

Solution: Check your internet connection and firewall settings. You can also download the appropriate driver manually from chromedriver.chromium.org and specify its path in your code.

2. Browser opens but test fails to interact with elements

Solution: Use explicit waits to ensure elements are fully loaded before interacting with them:

from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

element = WebDriverWait(driver, 10).until(
    EC.element_to_be_clickable((By.ID, "element-id"))
)
element.click()

3. Tests work locally but fail in CI/CD pipeline

Solution: Use headless mode for CI/CD environments:

from selenium.webdriver.chrome.options import Options

options = Options()
options.add_argument("--headless")
options.add_argument("--no-sandbox")
options.add_argument("--disable-dev-shm-usage")

driver = webdriver.Chrome(service=service, options=options)
